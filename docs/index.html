Taint Lite is a static taint-analysis tool that tracks Personally Identifiable Information through your code and reports every path where sensitive data reaches a dangerous sink — in IMP-Core and real Python.

Get Started View on GitHub
What it does

Everything you need to catch PII leaks

🔎

Forward Taint Analysis

Tracks sensitive data from sources (e.g. get_email()) through every assignment to sinks (e.g. log()).

🐍

Real Python Support

Analyzes actual .py files using Python's built-in AST — f-strings, dicts, chained assignments, function bodies, and more.

📋

External Policy Files

Define your own sources and sinks in a JSON file. Add domain-specific functions like kafka_publish or fetch_credit_card without touching the code.

💡

Actionable Recommendations

Every leak comes with a concrete fix: "Redact PII before logging", "Anonymize before analytics", etc.

Installation

Up and running in 60 seconds

Requirements: Python 3.9+ (no external runtime dependencies).

  1. Clone the repository
    $ git clone https://github.com/Justinv1234/cs336-p9-team9.git $ cd cs336-p9-team9

    $ pip3 install pytest

Usage

Running the analyzer

Basic syntax

$ python3 src/main.py <file.imp|file.py> [--policy policy.json]

The tool exits with code 0 if no leaks are found, or 1 if leaks are detected — suitable for use in CI pipelines.

Example 1 — IMP basic leak (examples/basic.imp)

examples/basic.imp
email := get_email()
name := get_name()
user_info := email + " - " + name
log(user_info)          # Privacy leak!
send_analytics(name)    # Another leak!
$ python3 src/main.py examples/basic.imp PII LEAK at line 4 Sink: log PII types exposed: {email, name} Recommendation: Redact PII before logging   PII LEAK at line 5 Sink: send_analytics PII types exposed: {name} Recommendation: Anonymize before sending to analytics

Example 2 — Python webapp leak (examples/webapp_leak.py)

examples/webapp_leak.py (excerpt)
email = get_email(user_id)
name  = get_name(user_id)
ssn   = get_ssn(user_id)

logging.info(f"Processing request for {email}")    # LEAK 1
send_analytics({"user": name, "action": "login"})  # LEAK 2
send_to_api("https://internal/verify", data={"ssn": ssn})  # LEAK 3
$ python3 src/main.py examples/webapp_leak.py PII LEAK at line 22 Sink: info   PII types exposed: {email} Recommendation: Redact PII before logging   PII LEAK at line 25 Sink: send_analytics   PII types exposed: {name} Recommendation: Anonymize before sending to analytics   PII LEAK at line 29 Sink: send_to_api   PII types exposed: {ssn} Recommendation: Remove PII before sending to external sink

Example 3 — No leak (examples/no_leak.imp)

$ python3 src/main.py examples/no_leak.imp No PII leaks detected.

Example 4 — Chained propagation (examples/chained.imp)

Taint follows data through a 3-variable chain — even indirect leaks are caught.

examples/chained.imp
email := get_email()
a := email
b := a
c := b + " extra"
log(c)              # leak: email reached log through a → b → c
$ python3 src/main.py examples/chained.imp PII LEAK at line 5 Sink: log   PII types exposed: {email} Recommendation: Redact PII before logging
External Policy

Customize sources & sinks with a policy file

The built-in policy covers common PII functions and sinks, but every application is different. Pass a JSON policy file via --policy to extend the defaults with your own sources and sinks.

⚡ Policy files merge with built-in defaults — you only need to list what you want to add or override.

Policy file schema

examples/custom_policy.json
{
  "sources": {
    "fetch_credit_card":   "credit_card",
    "get_passport_number": "passport",
    "load_biometric":      ["biometric", "sensitive"]
  },
  "sinks": {
    "kafka_publish": "Remove PII before publishing to Kafka",
    "s3_upload":     "Encrypt PII before uploading to S3",
    "slack_notify":  "Redact PII before sending to Slack"
  },
  "sink_patterns": [
    "log", "analytics", "api", "server",
    "publish", "upload", "notify", "export"
  ]
}

Using the policy

$ python3 src/main.py examples/custom_policy_demo.py --policy examples/custom_policy.json PII LEAK at line 18 Sink: kafka_publish   PII types exposed: {credit_card} Recommendation: Remove PII before publishing to Kafka   PII LEAK at line 21 Sink: s3_upload   PII types exposed: {passport} Recommendation: Encrypt PII before uploading to S3

Field reference

FieldTypeDescription
sourcesobjectMap of function_name → tag or function_name → [tag, ...]. Merged with defaults.
sinksobjectMap of function_name → recommendation_message. Merged with defaults.
sink_patternsarrayList of substrings — any function whose name contains one is treated as a sink. Replaces the default list when provided.
Python Support

Analyze real Python source files

Point the tool at any .py file. The Python frontend uses Python's built-in ast module to parse the code and runs the same taint engine as the IMP-Core analyzer.

Supported Python patterns

PatternExampleNotes

Variable propagationy = xTaint carried forward
String concatenations = a + bUnion of taints
f-stringsf"Hello {name}"Taint from embedded vars

Augmented assignmentmsg += nameAccumulates taint
